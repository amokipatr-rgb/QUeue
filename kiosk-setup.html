Electron Kiosk App Windows

No Chrome, no flags, no configuration needed. A dedicated kiosk app that runs fullscreen with autoplay built-in. Double-click and it works.

  1. Download Installer (77 MB)
  2. Run the downloaded QueueKiosk-Setup-1.0.0.exe as Administrator.
  3. Done! A "Queue Kiosk Display" shortcut appears on the Desktop and auto-starts on next boot.
Tip: After setup, configure the kiosk URL by setting the KIOSK_URL environment variable (e.g., http://localhost:5000/public-display.html for local, https://queue-production-2a11.up.railway.app/public-display.html for production).

Student Token Kiosk Windows

Self-service touchscreen kiosk for generating queue tokens. Students select an office, choose a service, and enter their details — no staff assistance needed.

  1. Download Student Kiosk (77 MB)
  2. Run the downloaded QueueKiosk-Student-Setup-1.0.0.exe as Administrator.
  3. Done! The student token kiosk appears fullscreen on the touchscreen and auto-starts on next boot.
Tip: On a touchscreen device, the OS virtual keyboard appears automatically when students tap the input fields. No additional setup needed.
1

Install Chrome Windows

If using the web version, install Chrome from google.com/chrome.

2

Run the Setup Script

Run the automated setup script to register Chrome autoplay policy and create a kiosk shortcut:

powershell -Command "iwr -Uri https://YOUR-RAILWAY-URL/setup-kiosk.ps1 -OutFile setup-kiosk.ps1; .\setup-kiosk.ps1"
3

Manual Chrome Setup

Create a Chrome shortcut with these flags:

"C:\Program Files\Google\Chrome\Application\chrome.exe" --autoplay-policy=no-user-gesture-required --kiosk --no-first-run --disable-features=TranslateUI https://YOUR-RAILWAY-URL/public-display.html
Place this shortcut in shell:startup for auto-launch.
1

Open in Chrome Android

Open the display URL in Chrome on your Android device or tablet.

https://YOUR-RAILWAY-URL/public-display.html
2

Enable Autoplay in Chrome Flags

In the Chrome address bar, go to:

chrome://flags/#autoplay-policy

Set "Autoplay policy" to "No user gesture is required", then tap Relaunch.

3

Add to Home Screen

Tap the menu (three dots) → Add to Home screen. This creates a fullscreen app-like icon. Open it and the display will fill the whole screen.

1

Install Chromium Linux

Install Chromium browser if not already installed:

sudo apt update && sudo apt install -y chromium-browser unclutter
2

Create Autostart Entry

Create a desktop autostart file that launches Chromium in kiosk mode:

mkdir -p ~/.config/autostart cat > ~/.config/autostart/queue-display.desktop << 'EOF' [Desktop Entry] Type=Application Name=Queue Display Exec=chromium-browser --noerrdialogs --disable-infobars --kiosk --autoplay-policy=no-user-gesture-required https://YOUR-RAILWAY-URL/public-display.html X-GNOME-Autostart-enabled=true EOF
On Raspberry Pi OS, the display will launch automatically on next boot. unclutter hides the mouse cursor.
1

Install Chrome macOS

Ensure Chrome is installed. Download from google.com/chrome.

2

Create Kiosk App with Automator

Open Automator → New → Application. Add "Run Shell Script" and paste:

open -a "Google Chrome" \ --args --autoplay-policy=no-user-gesture-required \ --kiosk --disable-features=TranslateUI \ https://YOUR-RAILWAY-URL/public-display.html
Save the Automator app, then add it to System Settings → General → Login Items.
3

Prevent Sleep

Go to System Settings → Energy Saver and set the display to never sleep.
